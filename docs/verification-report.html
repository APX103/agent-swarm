一、如何部署

三个角色，一台机器即可：

  1. Swarm 后端（本仓库）—— 1 个进程，端口 9000，对外只开这一个口。自带 Web UI（/ui）+ 所有 API。
  2. Redis —— Agent 注册中心，1 个容器。
  3. Agent 容器群 —— 你们团队开发的各个 Agent，每个一个容器，常驻运行，监听各自端口（9001、9002…）。

本次验证用的部署方式（裸机，后端跑宿主机）

# 1. 起 Redis
docker run -d --name swarm-redis -p 6379:6379 redis:7-alpine

# 2. 起后端（用项目 venv）
.venv/bin/python -m uvicorn src.main:app --host 127.0.0.1 --port 9000

后端启动时会自动：连 Redis → 初始化 5 个 worker 容器池 → 注册 agents/*.yaml 里的声明式 Agent → 挂载 Web UI。

健康检查确认就绪

$ curl http://127.0.0.1:9000/api/health
{"status":"ok","pool_available":5,"pool_total":5,"active_tasks":0}

四、得到的结果

编排器从零生成的 backend/main.py（完整、可直接运行）：

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel
from typing import List

app = FastAPI(title="Todo API")

app.add_middleware(CORSMiddleware, allow_origins=["*"],
    allow_credentials=True, allow_methods=["*"], allow_headers=["*"])

todos = [{"id": 1, "title": "示例", "done": False}]
next_id = 2

class TodoCreate(BaseModel):
    title: str

class Todo(BaseModel):
    id: int
    title: str
    done: bool

@app.get("/api/todos", response_model=List[Todo])
def get_todos():
    return todos

@app.post("/api/todos", response_model=Todo)
def create_todo(todo: TodoCreate):
    global next_id
    new_todo = {"id": next_id, "title": todo.title, "done": False}
    next_id += 1
    todos.append(new_todo)
    return new_todo

七、发现并修复的 Bug

Bug：worker 容器 SHARED_DIR 注入丢失 生产级

现象：worker 写文件成功（progress 日志证明），但产物落不到任务目录，编排器 review 找不到文件。

根因：pool 正确把 shared_dir 写进了 config.json，entrypoint 也 export 了 SHARED_DIR，但 worker.py main() 只解析 --role/--model/--base-url/--api-key/--port/--task-id，没有 --shared-dir。所以 worker 永远用模块顶层的默认值 /workspace/artifacts，config 里的任务路径被丢了。

修复（已落地，327 tests passed 无回归）：

  • src/agents/worker.py：加 --shared-dir 参数解析 + 设全局变量和环境变量
  • docker/entrypoint.sh：把 --shared-dir "$SHARED_DIR" 传给 worker.py

修复后第二次任务产物正确落盘到 session 目录 ✅。
