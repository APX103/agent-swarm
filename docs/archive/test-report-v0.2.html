📋 测试计划

本次测试覆盖 External Agent Adapter 功能的完整生命周期，包含 11 个测试模块、7 个 E2E 阶段，共 156 个测试用例。

1
E2E 阶段一：Agent 注册（Registration）
验证外部 Agent 的注册流程，包括 OpenAI、CLI、MCP 三种协议的注册，以及字段校验与事件触发。
  • 覆盖范围：6 个测试 — test_register_openai_agent, test_register_cli_agent, test_register_mcp_agent, test_register_missing_name, test_register_missing_endpoint, test_register_creates_online_events
  • 验证目标：多协议注册正确性、必填字段校验、在线事件自动生成
2
E2E 阶段二：心跳保活（Heartbeat）
验证 Agent 心跳机制，包括心跳成功刷新 TTL、时间戳更新、未知 Agent 的错误处理。
  • 覆盖范围：3 个测试 — test_heartbeat_success, test_heartbeat_updates_timestamp, test_heartbeat_unknown_agent
  • 验证目标：心跳接口正确性、TTL 续期、异常路径处理
3
E2E 阶段三：Agent 查询（Query）
验证已注册 Agent 的查询功能，包括列表查询、技能筛选、完整字段返回。
  • 覆盖范围：4 个测试 — test_list_all_agents, test_find_by_skill, test_find_by_skill_no_match, test_agent_has_complete_fields
  • 验证目标：列表接口完整性、技能过滤准确性、字段完备性
4
E2E 阶段四：Agent 调用（Invocation）
验证通过 API 调用外部 Agent 的完整流程，包括正常调用、缺失参数、Agent 不存在等场景。
  • 覆盖范围：4 个测试 — test_invoke_openai_agent, test_invoke_missing_task, test_invoke_agent_not_found, test_invoke_returns_result
  • 验证目标：调用链路完整性、错误处理、结果返回格式
5
E2E 阶段五：Agent 注销（Deregistration）
验证 Agent 注销流程，包括注销后心跳 404、列表中不可见、重复注销 404、离线事件触发。
  • 覆盖范围：5 个测试 — test_deregister_success, test_deregistered_heartbeat_404, test_deregistered_not_in_list, test_double_deregister_404, test_offline_event
  • 验证目标：注销彻底性、状态一致性、事件系统正确性
6
E2E 阶段六：TTL 过期（TTL Expiration）
验证 Agent TTL 过期后的自动清理机制，包括过期后心跳返回 404。
  • 覆盖范围：2 个测试 — test_expired_agent_removed, test_expired_agent_heartbeat_404
  • 验证目标：定时清理机制、过期 Agent 的状态隔离
7
E2E 阶段七：重新注册（Re-registration）
验证 Agent 重新注册的完整流程，包括新 Agent 注册、技能查询更新、连续心跳稳定性。
  • 覆盖范围：3 个测试 — test_reregister_new_agent, test_reregister_skill_query, test_consecutive_heartbeats
  • 验证目标：重新注册后状态一致性、技能更新、心跳稳定性
