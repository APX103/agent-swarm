二、本次修复的 bug

Bug 1（P0）：worker 写文件双层目录 backend/backend/main.py

根因：execute_file_tool 在 role_dir = shared_dir / "backend" 基础上再拼 LLM 传的 "backend/main.py"，导致 backend/backend/main.py。

修复：新增 _resolve_path()，如果 LLM 传的路径已含 role 前缀就不再重复拼接。

Bug 2（P0）：finalize review 找不到产物目录

根因：get_artifacts_dir(task_id) 读 task.work_dir，但 in-memory task 对象的 work_dir 有时没被正确设置，返回 None。

修复：新增 _get_artifacts_dir() 方法，task.work_dir 为空时回退到 _current_session_work_dir（worker 实际写文件的路径）。

三、调用链路

用户 POST /api/chat
  → Swarm 创建 task + session（work_dir = sessions/{sid}/）
  → orchestrator LLM tool-calling 循环：
    [1] GLM → plan_task（分析需求，拆子任务）
    [2] GLM → dispatch_agent(backend-engineer, "写 backend/main.py...")
        → Dispatcher → DockerBackend → pool.checkout → worker:9001
        → worker LLM 循环：write_file("backend/main.py") → _resolve_path → 正确路径
        → worker task completed → DockerBackend poll 检测到 → DispatchResult(success)
    [3] GLM → dispatch_agent(frontend-ux-pro, "写 frontend/index.html...")
        → 同上 → frontend/index.html 产出
    [4] GLM → finalize(summary)
        → _get_artifacts_dir() → session work_dir（回退生效）
        → review_artifacts: backend/ 有文件 ✅ frontend/ 有文件 ✅
        → review passed → task status → completed ✅
